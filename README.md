# О проекте

`radomir-ui` - веб-интерфейс `Радомир` для управления сервисом Radomir.
Интерфейс получает и отправляет данные только через binder-топики (`api/remote-topic.js`).

## Основные блоки интерфейса

- Панель режимов работы (`Фиксация`, `Следование`, `Выкл`)
- Область 3D-модели
- Боковая панель микрофона и распознанных голосовых команд
- Нижняя диагностическая панель статусов сервисов

## Технологический стек

- JavaScript (ECMAScript 2022+)
- Нативный DOM API
- CSS Grid/Flex
- Binder-топики по WebSocket

## Принципы реализации

- UI не выдумывает данные, а только отображает значения топиков.
- API-слой `api/*` не редактируется.
- Крупные UI-блоки реализованы отдельными компонентами-классами.
- Каждый компонент предоставляет `constructor`, `initialize` и `element`.
- Тяжелый 3D-компонент инициализируется один раз и переиспользуется.
- Структура репозитория поддерживается в `struct.md` и `ui-struct.md`.
