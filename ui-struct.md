# Структура UI

1. Корневой контейнер
- `.app` - общий контейнер приложения.
- `.main` - рабочая область интерфейса.

2. Основной двухколоночный макет
- `.model-panel` - общий контейнер макета.
- `.side-bar-host` - левая колонка под сайдбар тем.
- `.model-host` - правая колонка под 3D-модель.

3. Левый сайдбар
- `.side-bar` - стеклянная панель тем.
- `.side-bar-head` - верхняя часть панели.
- `.side-bar-title[data-side-bar-title]` - заголовок списка.
- `.topic-list[data-topic-list]` - прокручиваемый список тем.
- `template[data-topic-item-template]` - HTML-шаблон элемента списка тем.
- `.topic-item` - блок отдельной темы.
- `.topic-button[data-card-id]` - кнопка раскрытия темы.
- `.topic-title` - название темы.
- `.topic-marker` - визуальный индикатор раскрытия.
- `.topic-content` - раскрытая часть элемента.
- `.topic-description` - текст темы.
- `.side-bar-status[data-side-bar-status]` - статус загрузки или ошибки.

4. Область модели
- `.model-view` - контейнер 3D-карточки.
- `.model-status` - состояние загрузки или ошибки поверх сцены.

5. UI-состояния
- `.is-open` - раскрытый элемент списка тем.
- `.is-loading` - элемент в процессе загрузки.
- `.is-error` - ошибка загрузки.
- `.is-dragging` - пользователь вращает сцену.
