# Структура UI

1. Корневой контейнер
- `.app` - общий контейнер приложения.
- `.main` - рабочая область интерфейса.

2. Область модели
- `.model-panel` - панель с моделью и HUD.
- `.model-view` - контейнер 3D-модели.

3. HUD-слой
- `.hud-layer` - поверхностный слой управления.
- `.hud-top` - зона верхней панели.
- `.modes-panel` - переключатель режимов.
  - `.mode-button[data-mode="fix"]`
  - `.mode-button[data-mode="following"]`
  - `.mode-button[data-mode="off"]`

4. Боковая панель голосового ввода
- `.side-bar` - панель микрофона и команд.
- `.side-bar-head` - заголовочная часть.
  - `.mic-button[data-mic-toggle]` - кнопка микрофона.
  - `.wave` - визуализация распознавания.
- `.commands[data-commands]` - список распознанных фраз.
- `.command-item` - отдельная команда.

5. Диагностика
- `.diag-panel[data-diag-panel]` - нижняя диагностическая панель.
- `.diag-empty` - состояние отсутствия соединения.
- `.diag-items` - контейнер диагностических элементов.
- `.diag-item[data-service]` - элемент состояния сервиса.
  - `.diag-dot` - индикатор состояния.
  - `.diag-name` - имя сервиса.

6. UI-состояния
- `.is-active` - активная кнопка или элемент.
- `.is-mic-enabled` - микрофон включен.
- `.is-recognizing` - идет распознавание.
- `.is-disconnected` - элемент отключен.
- `.is-connected` - элемент подключен.
- `.is-empty` - в диагностике нет активных сервисов.
- `.is-loading` - модель в процессе загрузки.
- `.is-error` - ошибка загрузки модели.
